# 停产式迁移：Next.js 单体 → 三端架构

**决策**：停产当前 Next.js 单体，从零构建 Taro 小程序 + Express 后台 + React Admin 三端，全部就绪后一次性切换上线。不并行运行。

**为什么**：
- 无登录系统导致历史数据全部在 localStorage，无法迁移——停产不会丢失可恢复的数据
- 三个端从零构建允许一致的领域模型和技术栈，不需要兼容旧 API 格式
- 并行运行（爬行式迁移）需要在两套系统间协调，而当前用户量小，停产影响可控

**放弃的替代方案**：
- **爬行式迁移**（后台服务端先建，小程序端并行运行，逐步切流量）：需要维护两套 API 契约、处理数据迁移、写兼容层。当前无登录系统意味着迁移收益为零——没有数据可迁。
- **剥离式迁移**（先把题库管理剥离成独立服务）：题库管理逻辑目前不存在于 Next.js 里（全在用户侧的 LLM 实时生成），剥离没有意义。
