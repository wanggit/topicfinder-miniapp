# LLM Prompt 存 MySQL，管理后台在线编辑

**决策**：所有 LLM 系统提示词存储在 MySQL 中，通过管理后台在线编辑，即时生效，无需重新部署。

**为什么**：
- 辅导引擎的教学策略（苏格拉底式引导深度、内容安全护栏、角色定义）需要频繁调优，硬编码会为每次微调引入发版成本
- 不同年级/学科可能需要不同的辅导风格——存库可以按知识点粒度绑定不同 Prompt
- 初始 Prompt 由 LLM 自生成、管理员审核——这个设置流程天然要求 Prompt 可编辑
- Prompt 变更与代码变更的发布周期不同——Prompt 迭代是运营行为，不是工程行为

**放弃的替代方案**：
- **硬编码在 Git 中**：每次 Prompt 微调需要 build→deploy，迭代太慢。当前 Next.js 的 `lib/prompts/` 就是这样做的，新架构需要更快的反馈循环。
